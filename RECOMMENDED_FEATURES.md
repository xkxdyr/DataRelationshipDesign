## 数据库连接与同步功能说明

### 14. 数据库连接管理
**描述**: 支持配置和管理数据库连接

**推荐理由**:
- 提供完整的数据库连接配置界面
- 支持连接测试，确保配置正确
- 支持保存多个连接配置，方便切换

**实现建议**:
- 配置项：主机、端口、用户名、密码、数据库名
- 支持连接测试功能
- 支持 SSL 连接选项（可选）
- 连接信息加密存储（安全考虑）
- 连接配置持久化到本地

**待确认**:
- 支持的数据库类型
- SSL 连接实现方式
- 密码加密存储方案

### 15. 数据库逆向工程
**描述**: 从现有数据库导入表结构

**推荐理由**:
- 设计文档 Phase 3 明确提到此功能
- 极大提升工具实用性，支持从现有数据库开始设计
- 减少手动录入的工作量

**实现建议**:
- Phase 1: 仅支持 MySQL
- 导入内容：表结构、列、主键、外键、索引
- 支持选择要导入的表
- 类型映射：数据库类型 ↔ 设计工具类型
- 导入后可编辑和继续设计

**待确认**:
- 是否需要导入数据（建议仅导入结构）
- 是否需要支持视图、存储过程等
- 增量更新策略

### 16. 数据库同步功能
**描述**: 将设计同步到实际数据库

**推荐理由**:
- 提供完整的设计→生产工作流程
- 减少手动执行 DDL 的出错概率
- 支持预览同步变更

**实现建议**:
- Phase 1: 单向同步（设计→数据库）
- 生成同步 DDL 脚本
- 支持预览变更内容
- 支持手动确认后执行
- 同步前备份建议

**待确认**:
- 双向同步需求
- 冲突解决策略
- 事务支持

## UI优化功能说明

### 17. 增强字段选择器显示
**描述**: 在关系编辑器等场景选择表和字段时，显示表名注释、字段约束及注释

**用户明确需求**:
- 选择表时显示表注释
- 选择字段时显示字段约束（PK、FK、UNIQUE、NOT NULL等）及注释
- 提升信息透明度，减少选择错误

**实现建议**:
- 在 RelationshipEditor 的表选择下拉框中显示表名+注释
- 在字段选择下拉框中显示字段名+类型+约束标识+注释
- 使用 Tooltip 显示完整注释信息

**待确认**:
- 显示格式规范
- 约束标识样式

### 18. 级联规则悬浮提示
**描述**: 在关系编辑器的更新/删除规则处添加悬浮提示

**用户明确需求**:
- CASCADE: 级联更新/删除，子表记录同步更新/删除
- SET NULL: 设为NULL，子表关联字段设为NULL
- RESTRICT: 限制删除，阻止父表删除
- NO ACTION: 无动作，类似RESTRICT
- SET DEFAULT: 设为默认值，子表关联字段设为默认值

**实现建议**:
- 使用 Ant Design Tooltip 组件
- 在 Select 组件的 options 中添加 description 字段
- 悬浮时显示规则的实际作用说明

**待确认**:
- 提示内容的精确表述
- 是否需要示例说明

### 19. SQL导入一键整理
**描述**: 导入 SQL 项目后一键自动整理表布局

**用户明确需求**:
- 导入 SQL 后自动计算最优表布局
- 关系连线自动排列，避免交叉
- 优化表节点位置关系

**实现建议**:
- 实现自动布局算法（如树形布局、力导向布局）
- 根据关系自动调整表位置
- 一键触发整理功能

**待确认**:
- 布局算法选择
- 是否支持手动微调

### 20. 可视化布局优化
**描述**: 优化画布布局，让关系展示更清晰

**用户明确需求**:
- 关系线自动避开经过的节点
- 减少关系线交叉
- 关系线样式更清晰易读

**实现建议**:
- 使用 React Flow 的 smoothstep 或 bezier 边类型
- 实现边的智能路径规划
- 添加关系线高亮/淡化效果

**待确认**:
- 路径规划算法复杂度
- 性能影响评估

### 21. 可视化版本对比
**描述**: 提供版本对比界面，快速查看不同版本间差异

**用户明确需求**:
- 可视化展示两个版本的差异
- 高亮显示新增、修改、删除的表/字段/关系
- 便于快速了解版本变更内容

**实现建议**:
- 左右对比视图或diff样式展示
- 差异分类：高亮/绿色（新增）、橙色（修改）、红色（删除）
- 支持选择任意两个版本进行对比

**待确认**:
- 对比粒度（表级/字段级）
- 差异展示形式

### 22. 数据模拟功能
**描述**: 根据表结构自动生成模拟数据

**用户明确需求**:
- 根据字段类型自动生成模拟数据
- 支持自定义数据规则（如指定姓名库、邮箱格式等）
- 支持配置生成数量
- 方便测试使用

**实现建议**:
- 根据数据类型生成对应格式数据（INT随机数、VARCHAR随机字符串、EMAIL符合格式等）
- 提供规则配置界面
- 支持导出模拟数据

**待确认**:
- 支持的数据类型范围
- 自定义规则实现方式

## 离线/本地模式功能说明

### 7. 离线数据存储
**描述**: 使用浏览器内置的 IndexedDB 或 LocalStorage 存储项目数据

**实现建议**:
- 使用 Dexie.js 或 idb 库封装 IndexedDB 操作
- 项目数据模型序列化为 JSON 存储
- 支持数据的增删改查操作

**待确认**:
- 存储数据结构设计
- 数据迁移策略
- 存储容量限制处理

### 8. 离线模式支持
**描述**: 在无网络连接时仍能继续工作

**实现建议**:
- 检测网络状态（navigator.onLine）
- 离线时使用本地数据操作
- 网络恢复时自动同步到服务器
- 冲突解决策略

**待确认**:
- 同步冲突处理方式
- 离线操作队列管理
- 数据一致性保证

### 9. 本地项目导入导出
**描述**: 支持将项目导出为本地文件

**用户明确需求**:
- 导出格式: JSON + SQL (完整项目 JSON + DDL 脚本)
- 导入来源: 支持 JSON 文件导入 + SQL DDL 脚本解析导入
- 存储位置: 同时支持浏览器下载和 IndexedDB 本地存储
- 质量标准: 工业级功能

**实现建议**:
- 导出为 JSON: 包含完整项目数据（表、列、关系、索引、版本）
- 导出为 SQL: 生成 MySQL DDL 脚本，包含 CREATE TABLE、INDEX、FOREIGN KEY
- 导入 JSON: 解析项目数据，验证完整性，创建新项目
- 导入 SQL DDL: 解析 CREATE TABLE 语句，提取表名、列名、类型、约束等
- 导入时验证数据完整性，检查必填字段、数据类型一致性
- 支持导入冲突处理: 覆盖/跳过/新建选项

**待确认**:
- 导出文件格式标准 ✅ (JSON + SQL)
- 导入冲突处理 ✅ (覆盖/跳过/新建)
- 大项目分片导出 - 小项目暂不需要

### 10. 自动保存到本地
**描述**: 定期自动保存设计数据

**实现建议**:
- 使用 debounce 防抖保存
- 保存频率配置化
- 保存状态提示

**待确认**:
- 自动保存间隔时间
- 保存失败重试策略
- 手动保存与自动保存冲突处理

### 11. 本地模式切换
**描述**: 用户可选择仅使用本地存储

**实现建议**:
- 界面提供模式切换开关
- 本地模式标识明显
- 在线/离线状态指示器

**待确认**:
- 模式切换对现有数据的影响
- 本地模式与服务器数据的交互
- 用户引导和提示

### 12. 本地数据库可视化
**描述**: 支持连接本地数据库

**实现建议**:
- 支持 SQLite 本地文件
- 读取现有数据库结构
- 生成 ER 图

**待确认**:
- 支持的本地数据库类型
- 安全连接方式
- 性能考虑
