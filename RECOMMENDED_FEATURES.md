## 离线/本地模式功能说明

### 7. 离线数据存储
**描述**: 使用浏览器内置的 IndexedDB 或 LocalStorage 存储项目数据

**实现建议**:
- 使用 Dexie.js 或 idb 库封装 IndexedDB 操作
- 项目数据模型序列化为 JSON 存储
- 支持数据的增删改查操作

**待确认**:
- 存储数据结构设计
- 数据迁移策略
- 存储容量限制处理

### 8. 离线模式支持
**描述**: 在无网络连接时仍能继续工作

**实现建议**:
- 检测网络状态（navigator.onLine）
- 离线时使用本地数据操作
- 网络恢复时自动同步到服务器
- 冲突解决策略

**待确认**:
- 同步冲突处理方式
- 离线操作队列管理
- 数据一致性保证

### 9. 本地项目导入导出
**描述**: 支持将项目导出为本地文件

**用户明确需求**:
- 导出格式: JSON + SQL (完整项目 JSON + DDL 脚本)
- 导入来源: 支持 JSON 文件导入 + SQL DDL 脚本解析导入
- 存储位置: 同时支持浏览器下载和 IndexedDB 本地存储
- 质量标准: 工业级功能

**实现建议**:
- 导出为 JSON: 包含完整项目数据（表、列、关系、索引、版本）
- 导出为 SQL: 生成 MySQL DDL 脚本，包含 CREATE TABLE、INDEX、FOREIGN KEY
- 导入 JSON: 解析项目数据，验证完整性，创建新项目
- 导入 SQL DDL: 解析 CREATE TABLE 语句，提取表名、列名、类型、约束等
- 导入时验证数据完整性，检查必填字段、数据类型一致性
- 支持导入冲突处理: 覆盖/跳过/新建选项

**待确认**:
- 导出文件格式标准 ✅ (JSON + SQL)
- 导入冲突处理 ✅ (覆盖/跳过/新建)
- 大项目分片导出 - 小项目暂不需要

### 10. 自动保存到本地
**描述**: 定期自动保存设计数据

**实现建议**:
- 使用 debounce 防抖保存
- 保存频率配置化
- 保存状态提示

**待确认**:
- 自动保存间隔时间
- 保存失败重试策略
- 手动保存与自动保存冲突处理

### 11. 本地模式切换
**描述**: 用户可选择仅使用本地存储

**实现建议**:
- 界面提供模式切换开关
- 本地模式标识明显
- 在线/离线状态指示器

**待确认**:
- 模式切换对现有数据的影响
- 本地模式与服务器数据的交互
- 用户引导和提示

### 12. 本地数据库可视化
**描述**: 支持连接本地数据库

**实现建议**:
- 支持 SQLite 本地文件
- 读取现有数据库结构
- 生成 ER 图

**待确认**:
- 支持的本地数据库类型
- 安全连接方式
- 性能考虑
