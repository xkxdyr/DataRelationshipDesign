## 需求模糊点

### 数据库连接与同步
- **数据库类型支持**：仅 MySQL 还是多数据库（MySQL、PostgreSQL、SQLite、SQL Server、Oracle）？
- **连接配置**：如何配置数据库连接信息（主机、端口、用户名、密码、数据库名）？
- **连接管理**：是否需要保存多个连接配置？是否需要连接测试功能？
- **同步模式**：单向（设计→数据库）还是双向（设计↔数据库）？
- **同步策略**：覆盖还是合并？冲突如何处理？
- **同步范围**：全库同步还是仅同步选中的表？
- **安全考虑**：是否需要 SSL 连接支持？密码如何安全存储？
- **性能优化**：是否需要连接池管理？大数据库如何处理？

### 数据库逆向工程
- **导入范围**：仅导入表结构，还是包含数据？
- **导入选项**：是否需要过滤表？是否需要选择导入的对象（表、视图、存储过程等）？
- **关系识别**：如何识别和导入外键关系？
- **索引导入**：如何处理索引和约束？
- **增量更新**：是否支持增量导入已有设计？
- **注释处理**：如何导入数据库注释？
- **类型映射**：数据库原生类型如何映射到设计工具的类型系统？

### 评论与标注
- 评论显示方式（内嵌评论还是独立面板）？
- 是否需要 @提及功能？
- 评论解决流程（开放/解决/关闭）？
- 评论导出？
- 导出报告（标记）？

## 已确认的功能决策（2026-05-16 讨论确认）

### 团队协作功能
**已确认决策**：

1. **个人项目协作**
- 支持个人项目邀请协作，但限制人数（如最多5人）
- 被邀请者权限仅为 editor 或 viewer
- 提供"个人项目 → 团队项目"一键转换功能

2. **团队项目协作**
- 大力支持，这是团队项目的核心价值

3. **锁机制**
- 锁粒度：表级锁 + 字段级锁
- 锁提示：显示"XX用户正在编辑"
- 锁规则：字段编辑锁字段、增删字段锁表、调整顺序锁表
- 锁超时：5分钟无操作自动释放

4. **操作日志**
- 所有操作留痕（Who/When/What/Where/Change）
- 日志维度：项目级/表级/字段级/关系级/索引级
- 存储：跟随项目，不自动释放
- 半年后提醒用户清理（以项目最后活动日期起算）
- 支持导入导出（JSON + CSV）

5. **协作布局**
- 自动布局（层级布局按依赖关系排列）
- 避免手动拖拽导致的冲突

6. **长连接**
- WebSocket + Yjs 实时同步

7. **大模型接入**
- 个人模型：个人接入，他人无法共享
- 团队模型：团队接入，成员共享
- 支持云端模型 + 本地部署模型
- API Key 加密存储（AES-256-GCM）
- AI操作必须经过用户确认
- 每次AI操作前自动创建快照
- 连接测试 + 恶意训练检测
- 向量库：双层架构（前端Voy.js + 后端pgvector）

8. **新增功能**
- 数据模拟生成 + 远程写入物理数据库
- 版本对比功能
- 可视化布局优化（关系线智能避让）

## 建议

### 数据库连接与同步功能
建议拆分为多个阶段实现：

- **Phase 1: 基础连接与逆向工程**
  - 仅 MySQL 支持
  - 配置简单连接信息（主机、端口、用户名、密码、数据库名）
  - 从现有数据库导入表结构（逆向工程）
  - 支持基本的表、列、主键、外键导入

- **Phase 2: 连接管理与同步**
  - 保存多个连接配置
  - 添加连接测试功能
  - 单向同步（设计→数据库）
  - 支持 DDL 执行

- **Phase 3: 高级功能**
  - 支持更多数据库（PostgreSQL、SQLite、SQL Server、Oracle）
  - 双向同步（设计↔数据库）
  - 冲突检测与解决
  - SSL 连接支持
  - 同步预览与确认

### 数据库逆向工程功能
建议与数据库连接功能一起实现，作为 Phase 1 的核心功能
