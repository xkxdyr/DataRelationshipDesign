## 一、技术选型（核心决策）

| 选型项 | 方案 | 理由 |
|--------|------|------|
| **WebSocket库** | `ws` | 轻量级、高性能、低延迟，比Socket.io少一层封装 |
| **序列化** | `msgpackr` | MessagePack格式，比JSON小30-50%，解析更快 |
| **CRDT引擎** | `yjs` | 去中心化冲突解决、自动合并、离线支持、O(1)冲突解决 |
| **身份认证** | JWT Token | 无状态、高性能、支持水平扩展 |
| **数据一致性** | 最终一致性 | 高性能、高可用、适合分布式协作 |

```typescript
enum MessageType {
  // 用户相关
  USER_JOIN = 'user:join',
  USER_LEAVE = 'user:leave',
  USER_LIST = 'user:list',

  // 操作相关
  OP_CREATE = 'op:create',
  OP_UPDATE = 'op:update',
  OP_DELETE = 'op:delete',

  // 同步相关
  SYNC_REQUEST = 'sync:request',
  SYNC_RESPONSE = 'sync:response',

  // 系统相关
  PING = 'system:ping',
  PONG = 'system:pong',
  ERROR = 'system:error',
}
```

## 七、性能优化目标

| 优化项 | 技术手段 | 预期收益 |
|--------|---------|---------|
| 序列化 | MessagePack | 30-50%带宽节省 |
| 增量同步 | CRDT | 减少90%数据量 |
| 批处理 | 操作合并 | 减少50%消息数 |
| 异步写入 | 非阻塞持久化 | 不影响实时性能 |
